# Report Delivery Guide

Use this guide to make the final output feel like a polished strategy report rather than a rough notes dump.

## Report formatting principles

- Start with a short executive summary.
- Prefer bullets over dense paragraphs.
- Use Markdown tables whenever the reader needs to compare options, priorities, or actions.
- Keep section labels explicit and predictable.
- Avoid giant unbroken lists of keywords.

## Recommended report flow

1. Executive summary
2. Site type diagnosis
3. Growth goal interpretation
4. Topic universe
5. Cluster architecture
6. Optional gap and inventory insights
7. Site-type-specific strategy
8. Execution roadmap

## Good use cases for tables

### Cluster table
Use a table for cluster-level comparison:
- cluster name
- intent
- pillar or hub page
- supporting pages
- business value
- priority

### Roadmap table
Use a table for sequencing:
- phase
- focus
- recommended outputs

### Gap or inventory table
Use a table when competitor or content inventory inputs are present:
- observed gap or asset
- recommended action
- rationale

## Preview workflow

When environment support exists for browser preview:
1. Convert the report into a temporary preview-friendly file.
2. Prefer HTML for readability when tables are substantial.
3. Open the preview URL for the user.
4. Keep the preview artifact temporary unless the user explicitly asks to save the report.

## Save workflow

After presenting or previewing the report:
1. Ask whether the user wants to save this report.
2. Save only on explicit confirmation.
3. Prefer a descriptive filename such as `topic-cluster-report-YYYY-MM-DD.md`.
4. Optionally save an HTML companion file when the rendered report matters.

## Fallback behavior

If preview tooling or URL generation is unavailable:
- present the report inline in chat
- keep formatting clean and scannable
- still ask whether the user wants the report saved as a file
